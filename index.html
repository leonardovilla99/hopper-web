Everything in one window

A conductor for your coding agents

Every task runs isolated, streams live, and lands as a reviewable diff — then commits to a branch and opens a PR.

Parallel worktrees

Each task gets its own isolated copy of the repo on its own branch. Run many agents at once — they never step on each other.

Plan → Build

Draft a plan and spec docs first, then switch to implementation. The UX step even renders a live HTML mockup of the component.

Compare & debate beta

Race the same task across models side-by-side and pick a winner — or let them debate: plan separately, critique each other, synthesize the best.

Verify → fix loop

Run your real checks (typecheck, lint, tests). Failures feed back to the agent automatically until they pass — or it stops and tells you.

Any model

Claude Code out of the box, plus any OpenAI-compatible endpoint — OpenRouter, Ollama, LM Studio, llama.cpp. Cloud and local, side by side.

Chat, @-mention & dictation

A read-only scratch chat, @-mention files for context, and local Whisper dictation — talk to your agents instead of typing.

Learning memory

Hopper reflects on what you merge and discard, learns your conventions, and injects them into future runs — getting more "you" over time.

Linear, commit & PR

Pull a ticket straight into a task, then commit to a ticket-named branch and open a GitHub PR — without leaving the app.

Test locally

Overlay any task's changes onto your real checkout to run them — then unload to a clean tree. Try a change before you keep it.

Built for keyboards

A ⌘K command palette, customizable hotkeys, queued prompts, slash commands, and a movable logs window. Fast by design.

Signature feature

Don't pick a model. Let them debate.

Turn "which approach is best?" into a structured argument. Each model plans independently, then reads and critiques the others — and a judge synthesizes the strongest single plan to move forward.
